SLA Compliance Tracking

SQL 
-- SLA Compliance Dashboard Queries

-- Current SLA status by severity
SELECT 
    v.severity,
    COUNT(*) as total_vulns,
    COUNT(*) FILTER (WHERE v.remediated_date IS NOT NULL) as remediated,
    COUNT(*) FILTER (WHERE v.remediated_date IS NULL 
                     AND v.sla_due_date > CURRENT_DATE) as within_sla,
    COUNT(*) FILTER (WHERE v.remediated_date IS NULL 
                     AND v.sla_due_date <= CURRENT_DATE) as sla_breached,
    ROUND(100.0 * COUNT(*) FILTER (WHERE v.remediated_date <= v.sla_due_date 
                                    OR (v.remediated_date IS NULL 
                                        AND v.sla_due_date > CURRENT_DATE))
          / COUNT(*), 1) as compliance_pct
FROM vulnerabilities v
WHERE v.status != 'Closed'
GROUP BY v.severity
ORDER BY 
    CASE v.severity 
        WHEN 'Critical' THEN 1 
        WHEN 'High' THEN 2 
        WHEN 'Medium' THEN 3 
        ELSE 4 
    END;

-- SLA breaches by team
SELECT 
    a.owner_team,
    COUNT(*) as breached_count,
    AVG(CURRENT_DATE - v.sla_due_date) as avg_days_overdue
FROM vulnerabilities v
JOIN assets a ON v.asset_id = a.asset_id
WHERE v.remediated_date IS NULL
  AND v.sla_due_date < CURRENT_DATE
GROUP BY a.owner_team
ORDER BY breached_count DESC;

-- Vulnerabilities approaching SLA (next 7 days)
SELECT 
    v.cve_id,
    a.hostname,
    v.severity,
    v.sla_due_date,
    v.sla_due_date - CURRENT_DATE as days_remaining,
    a.owner_team
FROM vulnerabilities v
JOIN assets a ON v.asset_id = a.asset_id
WHERE v.remediated_date IS NULL
  AND v.sla_due_date BETWEEN CURRENT_DATE AND CURRENT_DATE + 7
ORDER BY v.sla_due_date;

"How do you define and enforce vulnerability remediation SLAs?"

Sample answer: "I use a matrix based on vulnerability severity AND asset criticality. A critical vulnerability on a Tier 1 system gets 24 hours; the same vulnerability on a Tier 4 test server gets 14 days. SLA clock starts at detection, not ticket creation. I automate escalation: at 50% SLA I notify managers, at 75% directors get involved, and breaches go to VPs. I track SLA compliance as a KPIâ€”typically targeting 90%+ for critical vulnerabilities. SLA breaches require documented explanation and remediation plan."

Summary

SLA management provides accountability for vulnerability remediation. Define SLAs based on risk (severity + asset criticality), automate tracking and escalation, and report on compliance regularly. SLAs without enforcement become meaningless.
