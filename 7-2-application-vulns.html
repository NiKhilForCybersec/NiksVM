Software Composition Analysis

Python 
"""
Dependency Vulnerability Checking
Using safety (Python) and npm audit (Node.js).
"""

import subprocess
import json
from typing import List, Dict


def check_python_dependencies() -> List[Dict]:
    """Check Python dependencies for vulnerabilities using safety."""
    
    # Generate requirements
    subprocess.run(['pip', 'freeze', '>', 'requirements.txt'], shell=True)
    
    # Run safety check
    result = subprocess.run(
        ['safety', 'check', '--json', '-r', 'requirements.txt'],
        capture_output=True,
        text=True
    )
    
    if result.returncode != 0:
        vulns = json.loads(result.stdout)
        return [{
            'package': v[0],
            'installed_version': v[2],
            'vulnerability': v[3],
            'cve': v[4] if len(v) > 4 else None
        } for v in vulns]
    
    return []


def check_npm_dependencies(project_dir: str) -> List[Dict]:
    """Check Node.js dependencies for vulnerabilities."""
    
    result = subprocess.run(
        ['npm', 'audit', '--json'],
        capture_output=True,
        text=True,
        cwd=project_dir
    )
    
    audit_data = json.loads(result.stdout)
    vulns = []
    
    for advisory_id, advisory in audit_data.get('advisories', {}).items():
        vulns.append({
            'package': advisory['module_name'],
            'severity': advisory['severity'],
            'title': advisory['title'],
            'cve': advisory.get('cves', [None])[0],
            'patched_versions': advisory['patched_versions'],
            'recommendation': advisory['recommendation']
        })
    
    return vulns

"How do you handle dependency vulnerabilities in applications?"

Sample answer: "I use Software Composition Analysis tools integrated into CI/CD. Snyk or Dependabot scans on every PR, blocking merges for critical vulnerabilities. For production, I maintain a software inventory that maps applications to their dependencies so when something like Log4Shell hits, I can quickly identify affected systems. The challenge is transitive dependenciesâ€”your code might be fine but a library three levels deep is vulnerable. That's why automated scanning is essential."

Summary

Application vulnerabilities, especially in dependencies, are a massive attack surface. Implement Software Composition Analysis in your development pipeline, maintain a software inventory for rapid response, and prioritize widely-used libraries when critical CVEs emerge.
