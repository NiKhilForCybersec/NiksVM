Detection and Remediation

PowerShell 
# Windows: Check for missing patches
# Get installed hotfixes
Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 20

# Check specific KB
Get-HotFix -Id KB5001234

# Check Windows Update status
Get-WindowsUpdateLog

# Query for specific CVE via WMI
$CVE = "CVE-2021-34527"
Get-WmiObject -Query "SELECT * FROM Win32_QuickFixEngineering" | 
    Where-Object { $_.HotFixID -match "KB5004945|KB5004946" }
Bash 
#!/bin/bash
# Linux: Check for vulnerable packages

# RHEL/CentOS - Check for security updates
yum updateinfo list security

# Check specific CVE
yum updateinfo list --cve CVE-2021-4034

# Ubuntu/Debian - List upgradable security packages
apt list --upgradable 2>/dev/null | grep -i security

# Check if specific package is vulnerable
dpkg -l | grep polkit
apt-cache policy policykit-1

# Check kernel version
uname -r

# Check for known vulnerable kernel (Dirty Pipe)
KERNEL=$(uname -r | cut -d'-' -f1)
if [[ $(echo "$KERNEL 5.8" | awk '{print ($1 >= $2)}') -eq 1 ]] && \
   [[ $(echo "$KERNEL 5.16.11" | awk '{print ($1 < $2)}') -eq 1 ]]; then
    echo "Potentially vulnerable to Dirty Pipe (CVE-2022-0847)"
fi

Operating systems past End of Life (EOL) no longer receive security updates. Windows Server 2012 R2, Windows 7, CentOS 8, and Ubuntu 18.04 LTS are examples of EOL or approaching EOL systems that require migration planning. Scanners often flag EOL as critical vulnerabilities themselves.
