What is Vulnerability Management?

Imagine your company's network is a house. Vulnerability Management is like having a security guard who:
  • üîç Checks all the doors and windows (scanning)
  • üìã Makes a list of broken locks (identifying vulnerabilities)
  • üéØ Decides which to fix first - the front door before the attic window (prioritization)
  • üîß Fixes them or puts up extra security (remediation)
  • ‚úÖ Checks that the fixes worked (verification)

VM vs Penetration Testing

Quick Recall
Vulnerability Assessment = Doctor's Checkup (find all potential issues)
Penetration Testing = Exploratory Surgery (prove what's actually exploitable)
üîç Vulnerability Assessment
  • Automated scanning
  • Broad coverage (everything)
  • Finds weaknesses
  • Doesn't exploit
  • Continuous (weekly/monthly)
  • Answers: "What's vulnerable?"
üéØ Penetration Testing
  • Manual hacking
  • Deep, targeted (specific systems)
  • Proves exploitability
  • Actually exploits
  • Periodic (annual/quarterly)
  • Answers: "What can attacker achieve?"
üíº Interview Answer

Q: "What's the difference between VA and pentesting?"

"Vulnerability Assessment is like a doctor's full-body scan - automated checks that find all potential problems. Penetration testing is like exploratory surgery - a skilled expert manually attempts to exploit vulnerabilities to prove what an attacker could actually do. VA is broad and continuous; pentesting is deep and periodic. VA tells you what's vulnerable, pentesting tells you what's exploitable."

The CIA Triad in VM Context

C.I.A. = Confidentiality, Integrity, Availability

When assessing a vulnerability's impact, ask:

üîí C - Can attackers steal data? (Confidentiality breach)
‚úèÔ∏è I - Can attackers change data? (Integrity breach)
üíÄ A - Can attackers break the system? (Availability breach)

Ransomware hits all three = catastrophic! Data exfil = C only. DDoS = A only.

Common Acronyms - Must Know!

Acronyms Quick Reference
CVECommon Vulnerabilities & Exposures - unique vuln ID (CVE-2021-44228)
CVSSCommon Vulnerability Scoring System - severity 0-10
CWECommon Weakness Enumeration - vuln category/type
CPECommon Platform Enumeration - product identifier
NVDNational Vulnerability Database - US gov database
EPSSExploit Prediction Scoring System - likelihood of exploit
KEVKnown Exploited Vulnerabilities - CISA's "fix now" list
üß† CVE vs CWE vs CPE - Don't Confuse Them!

CVE = The specific vulnerability instance (CVE-2021-44228 = Log4Shell)
CWE = The type/category of weakness (CWE-502 = Deserialization)
CPE = The affected product (cpe:2.3:a:apache:log4j:2.14.1)

Think of it like a hospital:
CVE = "Patient John has the flu" (specific instance)
CWE = "Respiratory illness" (category of disease)
CPE = "John Smith, Age 35, Room 401" (identifying info)
