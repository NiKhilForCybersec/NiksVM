This page covers Microsoft Defender Vulnerability Management (MDVM) including integration with Defender for Endpoint, software inventory, security recommendations, browser extension vulnerabilities, and remediation workflows in the Microsoft security ecosystem.

Defender Vulnerability Management Overview

Microsoft Defender Vulnerability Management (formerly Threat and Vulnerability Management/TVM) is built into Microsoft Defender for Endpoint, providing agentless vulnerability assessment for devices already running the Defender sensor.

Key Capabilities

Capability Description
Software Inventory Automatic discovery of installed applications and versions
Vulnerability Assessment CVE detection based on software inventory
Security Recommendations Prioritized remediation guidance
Browser Extensions Visibility into browser add-ons and their risks
Certificate Assessment Track certificate expiration and issues
Secure Score Integration VM findings impact Microsoft Secure Score
ğŸ’œ Licensing Note

Basic MDVM is included with Defender for Endpoint P2. Advanced features like authenticated network scanning and premium vulnerability data require the Defender Vulnerability Management add-on license.

Software Inventory

MDVM automatically maintains a real-time software inventory across all Defender-protected endpoints without additional scanning.

KQL 
// KQL Query: Software Inventory Analysis
// Run in Microsoft 365 Defender Advanced Hunting

// Get all software with vulnerabilities
DeviceTvmSoftwareInventory
| where SoftwareName has "Adobe" or SoftwareName has "Java"
| summarize DeviceCount = dcount(DeviceId) by SoftwareName, SoftwareVersion
| order by DeviceCount desc

// Find devices running vulnerable software versions
DeviceTvmSoftwareVulnerabilities
| where CveId == "CVE-2023-44487"  // HTTP/2 Rapid Reset
| project DeviceName, SoftwareName, SoftwareVersion, CveId, VulnerabilitySeverityLevel
| order by VulnerabilitySeverityLevel desc

// Software with critical vulnerabilities
DeviceTvmSoftwareVulnerabilities
| where VulnerabilitySeverityLevel == "Critical"
| summarize 
    VulnCount = count(),
    DeviceCount = dcount(DeviceId) 
    by SoftwareName, SoftwareVersion
| order by VulnCount desc

// End-of-life software
DeviceTvmSoftwareInventory
| where EndOfSupportStatus == "EOS" or EndOfSupportStatus == "Upcoming EOS"
| summarize DeviceCount = dcount(DeviceId) by SoftwareName, SoftwareVersion, EndOfSupportDate
| order by EndOfSupportDate asc

"How does Microsoft Defender VM compare to traditional scanners like Nessus?"

Sample answer: "MDVM is agent-based using the existing Defender sensor, so there's no additional deployment. It provides continuous assessment without network scanning overhead. The integration with Microsoft's ecosystemâ€”Intune for patching, Sentinel for correlation, Secure Score for trackingâ€”makes it powerful for Microsoft-centric environments. However, it's limited to Defender-protected endpoints and doesn't cover network devices, Linux servers (without Defender for Linux), or OT systems. I'd use it as a complement to, not replacement for, traditional scanners in heterogeneous environments."

Summary

Microsoft Defender Vulnerability Management provides integrated, agentless vulnerability assessment for organizations already using Defender for Endpoint. Its tight integration with the Microsoft security ecosystem enables streamlined remediation workflows.
