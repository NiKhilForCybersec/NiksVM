Qualys Web Application Scanning (WAS)

Python 
"""
Qualys WAS API Integration
"""

import requests
from base64 import b64encode


class QualysWAS:
    def __init__(self, username: str, password: str, platform: str = 'qg2'):
        self.base_url = f'https://qualysapi.{platform}.apps.qualys.com/qps/rest/3.0'
        self.auth = b64encode(f'{username}:{password}'.encode()).decode()
        self.headers = {
            'Authorization': f'Basic {self.auth}',
            'Content-Type': 'application/xml'
        }
    
    def create_web_app(self, name: str, url: str) -> str:
        """Register a web application."""
        xml_data = f'''
        
            
                
                    {name}
                    {url}
                
            
        
        '''
        response = requests.post(
            f'{self.base_url}/create/was/webapp',
            headers=self.headers,
            data=xml_data
        )
        return response.text
    
    def launch_scan(self, webapp_id: str, scan_type: str = 'VULNERABILITY') -> str:
        """Launch a WAS scan."""
        xml_data = f'''
        
            
                
                    API Scan {webapp_id}
                    {scan_type}
                    
                        {webapp_id}
                    
                
            
        
        '''
        response = requests.post(
            f'{self.base_url}/launch/was/wasscan',
            headers=self.headers,
            data=xml_data
        )
        return response.text
    
    def get_scan_status(self, scan_id: str) -> str:
        """Get scan status."""
        response = requests.get(
            f'{self.base_url}/status/was/wasscan/{scan_id}',
            headers=self.headers
        )
        return response.text
    
    def get_findings(self, webapp_id: str) -> str:
        """Get vulnerability findings for a web app."""
        response = requests.get(
            f'{self.base_url}/search/was/finding',
            headers=self.headers,
            params={'webAppId': webapp_id}
        )
        return response.text


# Example
was = QualysWAS('username', 'password')
webapp = was.create_web_app('My App', 'https://myapp.example.com')
scan = was.launch_scan(webapp_id='12345')

CI/CD Integration

YAML 
# GitLab CI - Web Application Security Pipeline
stages:
  - build
  - sast
  - deploy-staging
  - dast
  - deploy-production

# SAST - Static Analysis
sast:
  stage: sast
  image: sonarsource/sonar-scanner-cli
  script:
    - sonar-scanner 
        -Dsonar.projectKey=$CI_PROJECT_NAME
        -Dsonar.sources=.
        -Dsonar.host.url=$SONAR_HOST
        -Dsonar.login=$SONAR_TOKEN
  allow_failure: true

# Deploy to staging for DAST
deploy-staging:
  stage: deploy-staging
  script:
    - kubectl apply -f k8s/staging/

# DAST - Dynamic Analysis
dast:
  stage: dast
  image: owasp/zap2docker-stable
  variables:
    DAST_TARGET_URL: https://staging.example.com
  script:
    # Wait for staging to be ready
    - |
      for i in {1..30}; do
        if curl -s -o /dev/null -w "%{http_code}" $DAST_TARGET_URL | grep -q "200"; then
          break
        fi
        sleep 10
      done
    # Run DAST scan
    - zap-baseline.py 
        -t $DAST_TARGET_URL 
        -r zap-report.html 
        -I  # Don't fail on warnings
    # Run full scan on main branch
    - |
      if [ "$CI_COMMIT_BRANCH" == "main" ]; then
        zap-full-scan.py -t $DAST_TARGET_URL -r zap-full-report.html
      fi
  artifacts:
    paths:
      - zap-report.html
      - zap-full-report.html
    when: always

# Deploy to production after DAST passes
deploy-production:
  stage: deploy-production
  script:
    - kubectl apply -f k8s/production/
  only:
    - main
  when: manual

"How do you implement web application vulnerability scanning in a DevSecOps pipeline?"

Sample answer: "I implement multiple layers: SAST in the IDE with SonarLint for immediate feedback, SAST in CI with SonarQube analyzing every commit, dependency scanning with Snyk or Dependabot, and DAST against staging environments. For DAST, I use ZAP baseline scans on every PR and full scans weekly. I configure authentication properly so the scanner tests authenticated functionality. Critical and high findings block deployment, while medium findings create tickets for the next sprint. I also ensure scan credentials are stored securely and results are integrated into our vulnerability management platform."

Summary

Web application security testing requires a combination of SAST, DAST, and potentially IAST for comprehensive coverage. Configure scanners properly with authentication, integrate into CI/CD pipelines for continuous testing, and pay special attention to API security for modern applications. The OWASP Top 10 provides a baseline for vulnerability categories that scanners should detect.
