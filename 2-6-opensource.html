Open Source Vulnerability Scanners

This page covers open source vulnerability scanning tools including OpenVAS/Greenbone, Nuclei for template-based scanning, Trivy for containers, Nmap NSE scripts, and OWASP ZAP for web applications.

OpenVAS / Greenbone

OpenVAS (Open Vulnerability Assessment Scanner) is the most comprehensive open source network vulnerability scanner. Greenbone provides the commercial-supported version.

Installation with Docker

Bash 
# OpenVAS/Greenbone Community Edition Docker Setup

# Create directories
mkdir -p /opt/gvm/data
cd /opt/gvm

# Docker Compose file
cat > docker-compose.yml << 'EOF'
version: '3.8'
services:
  gvm:
    image: greenbone/community-container
    container_name: greenbone-community
    hostname: greenbone
    ports:
      - "9392:9392"   # GSA Web Interface
      - "5432:5432"   # PostgreSQL (optional)
    volumes:
      - gvm-data:/var/lib/gvm
      - gvm-scap:/var/lib/gvm/scap-data
      - gvm-cert:/var/lib/gvm/cert-data
      - gvm-data-objects:/var/lib/gvm/data-objects
      - psql-data:/var/lib/postgresql
      - run-ospd:/run/ospd
    environment:
      - PASSWORD=admin  # Change this!
    restart: unless-stopped

volumes:
  gvm-data:
  gvm-scap:
  gvm-cert:
  gvm-data-objects:
  psql-data:
  run-ospd:
EOF

# Start OpenVAS
docker-compose up -d

# Wait for initialization (15-30 minutes first time)
docker-compose logs -f gvm

# Access web interface
# https://localhost:9392
# Default: admin / admin (change immediately)

Nuclei

Nuclei is a fast, template-based vulnerability scanner from ProjectDiscovery. It excels at web vulnerability scanning with a massive community-contributed template library.

Bash 
# Install Nuclei
go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest

# Or download binary
wget https://github.com/projectdiscovery/nuclei/releases/latest/download/nuclei_linux_amd64.zip
unzip nuclei_linux_amd64.zip
sudo mv nuclei /usr/local/bin/

# Update templates
nuclei -update-templates

# Basic scan against single target
nuclei -u https://target.com

# Scan with specific severity
nuclei -u https://target.com -s critical,high

# Scan multiple targets from file
nuclei -l targets.txt -o results.txt

# Scan with specific template tags
nuclei -u https://target.com -tags cve,rce,lfi

# Scan for specific CVE
nuclei -u https://target.com -t cves/2021/CVE-2021-44228.yaml

# Scan with rate limiting (requests per second)
nuclei -u https://target.com -rl 100

# Output formats
nuclei -u https://target.com -json -o results.json
nuclei -u https://target.com -me output_dir/  # Markdown export

# Scan entire subnet (with httpx for discovery)
echo "192.168.1.0/24" | httpx -silent | nuclei -t cves/

Custom Nuclei Templates

YAML 
# Custom Nuclei Template Example
# File: custom-checks/exposed-admin.yaml

id: exposed-admin-panel

info:
  name: Exposed Admin Panel Detection
  author: security-team
  severity: high
  description: Detects exposed administrative interfaces
  tags: admin,panel,exposure

requests:
  - method: GET
    path:
      - "{{BaseURL}}/admin"
      - "{{BaseURL}}/administrator"
      - "{{BaseURL}}/wp-admin"
      - "{{BaseURL}}/admin/login"
      - "{{BaseURL}}/manage"
      - "{{BaseURL}}/dashboard"
    
    matchers-condition: or
    matchers:
      - type: word
        words:
          - "Admin Login"
          - "Administrator"
          - "Dashboard"
          - "Control Panel"
        condition: or
        
      - type: status
        status:
          - 200
          - 302

# CVE Detection Template Example
# File: custom-checks/log4j-detect.yaml

id: log4j-rce-detect

info:
  name: Log4j RCE (CVE-2021-44228)
  author: security-team
  severity: critical
  description: Detects Log4j RCE vulnerability
  reference:
    - https://nvd.nist.gov/vuln/detail/CVE-2021-44228
  tags: cve,cve2021,rce,log4j,critical

requests:
  - raw:
      - |
        GET / HTTP/1.1
        Host: {{Hostname}}
        User-Agent: ${jndi:ldap://{{interactsh-url}}/a}
        X-Forwarded-For: ${jndi:ldap://{{interactsh-url}}/a}
        
    matchers:
      - type: word
        part: interactsh_protocol
        words:
          - "dns"
          - "http"

Trivy

Trivy is Aqua Security's comprehensive scanner for containers, filesystems, Git repositories, and infrastructure as code.

Bash 
# Install Trivy
# Debian/Ubuntu
sudo apt-get install wget apt-transport-https gnupg lsb-release
wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | sudo apt-key add -
echo deb https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main | sudo tee /etc/apt/sources.list.d/trivy.list
sudo apt-get update && sudo apt-get install trivy

# Scan container image
trivy image nginx:latest
trivy image --severity HIGH,CRITICAL alpine:3.18

# Scan local filesystem
trivy fs /path/to/project
trivy fs --scanners vuln,secret,config .

# Scan Git repository
trivy repo https://github.com/org/repo

# Scan Kubernetes cluster
trivy k8s --report summary cluster

# Scan IaC (Terraform, CloudFormation, etc.)
trivy config ./terraform/
trivy config --severity HIGH,CRITICAL ./cloudformation/

# Output formats
trivy image -f json -o results.json nginx:latest
trivy image -f sarif -o results.sarif nginx:latest  # For GitHub integration
trivy image -f template --template "@contrib/html.tpl" -o report.html nginx:latest

# Scan with ignore file
cat > .trivyignore << EOF
CVE-2023-12345
CVE-2023-67890
EOF
trivy image --ignorefile .trivyignore myimage:latest

# CI/CD Integration - fail on high/critical
trivy image --exit-code 1 --severity HIGH,CRITICAL myimage:latest

Nmap NSE Scripts

Nmap's Scripting Engine (NSE) includes vulnerability detection scripts that complement dedicated scanners.

Bash 
# Nmap Vulnerability Scanning

# Run all vuln scripts
nmap --script vuln 192.168.1.1

# Specific vulnerability checks
nmap --script smb-vuln-* 192.168.1.1           # SMB vulnerabilities
nmap --script ssl-* 192.168.1.1                 # SSL/TLS issues
nmap --script http-vuln-* 192.168.1.1          # HTTP vulnerabilities

# Common CVE checks
nmap --script smb-vuln-ms17-010 192.168.1.0/24  # EternalBlue
nmap --script http-vuln-cve2017-5638 target     # Apache Struts

# Safe vulnerability scanning (won't crash services)
nmap --script "vuln and safe" 192.168.1.1

# Comprehensive scan with version detection
nmap -sV --script=vuln -p- 192.168.1.1

# Output to multiple formats
nmap --script vuln -oA vuln_scan 192.168.1.0/24

# Update NSE scripts
nmap --script-updatedb

# List available vuln scripts
ls /usr/share/nmap/scripts/ | grep vuln

# Custom script execution
nmap --script /path/to/custom-check.nse 192.168.1.1

Useful NSE Vulnerability Scripts

Script Purpose

ssl-heartbleed Heartbleed vulnerability

OWASP ZAP

OWASP ZAP (Zed Attack Proxy) is the leading open source web application security scanner for DAST (Dynamic Application Security Testing).

Bash 
# ZAP Docker Usage

# Quick baseline scan
docker run -t zaproxy/zap-stable zap-baseline.py -t https://target.com

# Full scan
docker run -t zaproxy/zap-stable zap-full-scan.py -t https://target.com

# API scan
docker run -t zaproxy/zap-stable zap-api-scan.py -t https://api.target.com/openapi.json

# Save report
docker run -v $(pwd):/zap/wrk/:rw -t zaproxy/zap-stable zap-baseline.py \
    -t https://target.com \
    -r report.html

# ZAP CLI with custom config
docker run -t zaproxy/zap-stable zap-baseline.py \
    -t https://target.com \
    -c config.prop \
    -d  # Show debug output

# ZAP Daemon mode for automation
docker run -u zap -p 8080:8080 -d zaproxy/zap-stable \
    zap.sh -daemon -host 0.0.0.0 -port 8080 \
    -config api.addrs.addr.name=.* \
    -config api.addrs.addr.regex=true

# Use ZAP API
curl "http://localhost:8080/JSON/spider/action/scan/?url=https://target.com"
curl "http://localhost:8080/JSON/ascan/action/scan/?url=https://target.com"

Open Source Scanner Comparison

Tool Best For Strengths Limitations
OpenVAS Network vulnerability scanning Comprehensive, enterprise-like features Resource intensive, complex setup
Nuclei Web/API scanning, custom checks Fast, huge template library, easy custom templates Primarily web-focused
Trivy Containers, IaC, SBOM Fast, CI/CD friendly, multiple scan types No network scanning
Nmap NSE Network discovery + basic vuln checks Lightweight, excellent discovery Limited vuln coverage
OWASP ZAP Web application DAST Full-featured DAST, active community Web-only, can be slow
üí° Interview Question

"When would you use open source scanners instead of commercial tools?"

Sample answer: "Open source scanners have specific strengths: Trivy is excellent for container scanning in CI/CD pipelines‚Äîit's faster and more developer-friendly than most commercial options. Nuclei excels at custom vulnerability checks with its template system. I use them to complement commercial scanners: Trivy in our build pipeline, Nuclei for custom web checks, and ZAP for developer self-service DAST. For enterprise VM programs requiring compliance reporting, SLA tracking, and support, I still rely on commercial tools, but open source fills important gaps."

Summary

Open source vulnerability scanners provide powerful capabilities for specific use cases. Trivy dominates container scanning, Nuclei excels at web vulnerability detection with custom templates, and OpenVAS offers comprehensive network scanning. Use them strategically to complement commercial tools.
